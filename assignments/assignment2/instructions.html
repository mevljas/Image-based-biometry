Ear Detection Assignment

In the first assignment you merely used an existing detector. Here you will be training YOLOv8 for ear prediction. All the code this time is set up for you, you just need to optimize hyperparameters (and the augmentation parameters). Clone needed files and ear datasets from here into your GDrive. Copy the main Notebook for Colab from here into your own Colab project.

Prerequisites:

  1. Start a Google Colab project using the IBB code from above. If you have a CUDA-compatible GPU, you are welcome to use your own machine.
  2. In Google Colab, select Runtime > Change runtime type > T4 GPU.
  3. Import YOLOv8 files into the project from your Google Drive by executing the first code cell (modify paths as needed).
  4. Install project dependencies by running the second code cell.
  5. Attempt to train the model to verify that the code functions and data is being read. Do not expect best convergence at this stage as hyperparameters are left at default and the model lacks sufficient training data. Additionally, the first execution of train.py may take an extended period due to Colab scanning all images.
  6. All the outputs of the training and testing will be available in runs/detect/.

Tasks to Complete (Quantitative and Qualitative Evaluation):

  1. In train.py, fine-tune hyperparameters (different optimizer, learning rates, dropout levels, etc.), explanations and list of all the possible parameters at: https://docs.ultralytics.com/usage/cfg/#train. Additionally the yolov8m is selected in the script, feel free to try others. Store the best performing model from this stage.
  2. In train.py, configure data augmentation to suit the dataset, while using the best hyperparameters you have found so far. This step is important for effective model training. To do this, set the train parameter augment to True, and adjust the following train parameters: hsv_h, hsv_s, hsv_v, degrees, translate, scale, shear, perspective, flipud, fliplr, mosaic, mixup, copy_paste. Store the best performing model from this stage.
  3. Aim to achieve the highest possible mAP (mean Average Precision), by observing mAP50-95. Monitor the mAP50-95 on the validation set, which is already output by the code. In the en run evaluate.py on the best two models from points 1 and 2 and report mAP50-95 on the test set.
  4. Select 5 images where the detector performs poorly and analyze the potential reasons for this underperformance. Sorting and identifying the least effective (or the most interesting) detections is part of your task. Use predict.py to visualize predictions of these images.
